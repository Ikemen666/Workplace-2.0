# CEO-Level Strategic Briefing Template

This template outlines the structure and quality gates for producing a decision-driving memo for a C-level executive, based on the `hv-analysis` framework.

## Structure

1.  **Cover Page**
2.  **Executive Summary (1 page):** Must include a bullet on the economic headline (e.g., "MVP pays back in N weeks") and any key risk mitigations (e.g., "MVP scoped to contain JV precedent risk").
3.  **Section 1: How We Got Here (Vertical Axis):** Narrative history. Must plant specific, named "hooks" that will be paid off in the crossover section.
4.  **Section 2: What Peers Are Doing (Horizontal Axis):** Opinionated structure. Deep-dives on the most relevant 1-2 peers, concise summaries on others. Must include a master comparison table.
5.  **Section 3: The Argument (Crossover):** The core of the brief. Must include:
    *   **3a. Crossover:** Pays off the hooks from Section 1.
    *   **3b. The "Strawman" MVP:** A concrete, time-boxed, low-cost pilot proposal.
    *   **3c. The Economic Case:** Must include sourced baseline CAC, projected MVP CAC, gross profit per sale (as a range), and a payback period sensitivity analysis (bear/base/bull). Every number must be sourced or an explicitly anchored `[directional]` hypothesis.
    *   **3d. Scalable Solution to Core Problem:** A specific, mechanistic solution for the key operational challenge (e.g., the anti-`飞单` mechanism).
    *   **3e. Compliance Framework:** A section addressing the key legal/compliance hurdles (e.g., PIPL) with specific article citations and draft language.
    *   **3f. Risk Analysis:** Expanded risk section with mitigations and in-flight failure detection "tripwires."
    *   **3g. Scenarios:** Three 12-month scenarios (bear/base/bull) with named triggers.
    *   **3h. Co-Pitch Framing:** A dedicated section on how to frame the pitch for other key stakeholders (e.g., the OEM lead).
6.  **Section 4: Operating Model:** Must include:
    *   **4a. Bundled Product:** Definition of the offer.
    *   **4b. IT Integration Architecture:** A credible diagram and data flow that explicitly names the client's existing systems.
    *   **4c. Full RACI Table.**
    *   **4d. Full Roadmap (MVP and Scale-up phases).**
7.  **Section 5: Recommendations for Decision:** A numbered list of prescriptive recommendations, not neutral questions. Each must include the recommendation, the "why," the "trade-off accepted," and the specific "decision to confirm."
8.  **Appendix A: Sources & Footnotes:** A full, numbered list of all sources with URLs and access dates.
9.  **Appendix B:** Ancillary material (e.g., deep-dive on a non-critical competitor).

## Quality Gates (Non-Negotiable)

*   **Word Count:** Meets specified floor (e.g., >12,000 words).
*   **Footnote Count:** Meets specified floor (e.g., >25).
*   **No Naked Numbers:** Every number in the economic case is either footnoted or an anchored `[directional]` tag.
*   **Correct Mechanisms:** Solutions (e.g., commission structure) are mechanically sound and correctly aligned with incentives.
*   **Systems Named:** IT architecture diagram explicitly names the client's real-world systems.
*   **Banned Phrases Scrubbed:** A final find-and-replace pass is run to remove all specified banned phrases.
*   **Restored Content:** Verify that no required content from previous versions was dropped.
*   **Secondary Audience Addressed:** Explicitly addresses the needs and likely objections of secondary stakeholders.
*   **Final Output Format:** Matches the user's specified format (e.g., `.docx`).
